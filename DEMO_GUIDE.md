# Инструкция для защиты проекта (Недели 1-8)
**Курс:** Advanced Backend & DevOps
**Проект:** Investment Intelligence Hub (Платформа анализа стартапов)

---

## ⭐️ Что было сделано (реализация требований 1-8 недель)

1. **Неделя 1-2 (Окружение и Docker):**
   - Настроен `docker-compose.yml` (База данных PostgreSQL + FastAPI приложение).
   - Написан `.env.example`. Приложение и БД запускаются одной командой.
   - Добавлен healthcheck для БД (приложение ждет поднятия базы перед стартом).

2. **Неделя 3 (Модели и БД):**
   - Разработана схема: `Startups`, `Investors`, `Investments` с правильными связями (Foreign Keys).
   - Подключен Alembic для миграций. Создана миграция с новыми полями (description, status, и др.).
   - Настроен скрипт `init_db.py` для тестирования логики.

3. **Неделя 4-5 (Сборщик данных - Collector):**
   - Улучшенный скрипт `enhanced_collector.py` имитирует парсинг реальных данных.
   - **Обход блокировок:** Случайная замена `User-Agent`, ротация прокси-серверов, адаптивные задержки (delay), и механизм автоматических повторов (retries).
   - **Объем данных:** Генерирует более 50 структурированных стартапов с инвестициями (Requirement недели 4 полностью выполнен).

4. **Неделя 6 (Data Processor & Normalizer):**
   - Файл `normalizer.py` использует регулярные выражения для извлечения сущностей.
   - Очищает и нормализует названия компаний, имена инвесторов и суммы финансирования.
   - Подсчитывает комплексную оценку качества данных в базе (Quality Score).

5. **Неделя 7-8 (REST API):**
   - Развернут FastAPI сервер (файл `app/main.py`).
   - Реализована **сложная пагинация, фильтрация и сортировка**.
   - Добавлен эндпоинт **Full-text search** (`/search`), который ищет сразу по всем сущностям (стартапы, инвесторы, инвестиции).
   - Генерация красивой автоматической документации (Swagger UI и ReDoc).

6. **Неделя 11 (CI/CD) - Забегая вперед:**
   - Настроен GitHub Actions (`.github/workflows/ci.yml`). При пуше в `main` автоматически прогоняются линтеры (`flake8`) и собирается Docker образ для проверки сборки.

## 🚀 Как демонстрировать проект преподавателю

### Шаг 1. Демонстрация кода и репозитория
1. Откройте репозиторий на GitHub. Покажите чистую структуру.
2. Откройте вкладку **Actions**. Покажите, что у вас настроен CI/CD пайплайн, и он успешно проходит (зеленая галочка).
3. Продемонстрируйте файл `enhanced_collector.py` — акцентируйте внимание на классах `ProxyManager` и `UserAgentManager`, покажите обработчик ошибок HTTP-запросов и задержки.
4. Продемонстрируйте файл `normalizer.py` — покажите регулярные выражения в методах `extract_dates` и `extract_numbers`.

### Шаг 2. Запуск проекта
Откройте терминал в папке проекта и выполните запуск:
```bash
docker-compose up -d --build
```
Объясните, что `docker-compose` автоматически собирает контейнер FastAPI и поднимает базу PostgreSQL, дожидаясь ее готовности.

### Шаг 3. Инициализация базы данных
Покажите, как вы заполняете БД (эмуляция реального парсинга):
```bash
docker-compose exec web python init_db.py
```
Скрипт должен показать лог создания стартапов (их должно быть $> 50$).

### Шаг 4. Демонстрация Swagger UI (Документация)
1. Откройте в браузере: http://localhost:8000/docs
2. Покажите преподавателю красивый интерфейс Swagger. Акцентируйте внимание на описаниях эндпоинтов.

### Шаг 5. Проверка функциональности (REST API)
Прямо в Swagger (или через Postman) сделайте несколько запросов:
1. **Эндпоинт `/startups`:** Выполните запрос `GET`, покажите ответ. Обратите внимание на блок `meta` с пагинацией (total, pages, и т.д.).
2. **Сортировка:** Сделайте запрос `GET /startups?sort_by=name&sort_order=desc`. Покажите, что API умеет сортировать.
3. **Глобальный Поиск (`/search`):** Сделайте запрос `GET /search?q=Tech` (Например, поиск слова "Tech"). Покажите, как API возвращает и стартапы, и инвесторов, и инвестиции в едином ответе.
4. **Статистика:** Выполните запрос `GET /statistics`. Покажите агрегацию данных из БД.

### Шаг 6. Запуск тестов (по желанию)
Чтобы доказать, что все требования API работают корректно:
```bash
docker-compose exec web python test_api.py
```
В терминале появится красивый отчет о пройденных тестах (зеленые ✅).
