Командное проектное задание
Курс: Advanced Backend & DevOps
Система контроля версий: GitHub (обязательное использование репозитория)
Сдача: Репозиторий GitHub + развернутые контейнеры + отчет о нагрузочном тестировании

Описание проекта
Каждая команда разрабатывает независимый Backend-сервис (Microservice), который решает задачу автоматизированного сбора, обработки и предоставления данных в конкретной предметной области.
Темы проектов (на выбор команды):
Мы выбрали эту систему для реализации:
Investment Intelligence Hub: Платформа анализа венчурных инвестиций и стартапов (Crunchbase).
Функциональные и DevOps-требования
Проект должен включать следующие компоненты:
Data Collector (Сборщик): Реализация парсинга (Web Scraping) или работы с внешними API. Обход блокировок, работа по расписанию (Cron/Celery).
Data Storage (Хранилище): Проектирование схемы БД (PostgreSQL). Сохранение данных без дубликатов. Использование миграций (Alembic/Liquibase).
Data Processor (Обработчик): Очистка текста, выделение ключевых слов, нормализация данных.
RESTful API: Эндпоинты для поиска, фильтрации и выгрузки данных. Подключение авто-документации (Swagger/OpenAPI).
Auth Service: Защита API с помощью JWT (JSON Web Tokens).
Containerization: Полная упаковка приложения в Docker (Dockerfile) и оркестрация через docker-compose.
CI/CD Pipeline: Автоматическая сборка, линтинг и тестирование при пуше в GitHub (GitHub Actions/GitLab CI).
Observability: Сбор метрик (Prometheus + Grafana) и централизованные логи.
Performance Testing: Отчет о производительности системы (JMeter/k6) — критически важно.
Security: Управление секретами (ENV-переменные), защита от инъекций.

Этапы проекта и критерии оценки(На данный момент мы находимся на 8 неделе) 
Неделя 1: Планирование и анализ  
Выбор темы проекта.
Анализ источников данных (сайтов/API) для парсинга.
Распределение ролей (Backend Dev, Data Engineer, DevOps).
Оценка: Четкость понимания источника данных и задач.
Неделя 2: Настройка окружения
Создание репозитория GitHub.
Настройка базового Docker окружения (App + DB).
Настройка .gitignore и структуры проекта.
Оценка: Успешный запуск контейнера «Hello World».
Неделя 3: Проектирование БД и Моделей 
Разработка ER-диаграммы для выбранной предметной области.
Реализация моделей (ORM) и системы миграций.
Оценка: Соответствие структуры БД реальным данным источника.
Неделя 4: Сбор данных (Collector) – Часть 1 
Реализация базового парсера/скрейпера.
Настройка сохранения «сырых» данных в БД.
Оценка: Наличие реальных данных в базе (минимум 50 записей).
Неделя 5: Сбор данных (Collector) – Часть 2 (10%)
Реализация обхода блокировок (User-Agents, Proxy, Delay).
Автоматизация запуска (Cron/Scheduler).
Обработка ошибок соединения.
Оценка: Стабильность работы модуля сбора данных.
Неделя 6: Обработка и Бизнес-логика (10%) 
Реализация модуля очистки и нормализации данных.
Выделение сущностей (даты, имена, теги).
Оценка: Качество структурированных данных.
Неделя 7: REST API – Базовый уровень (5%)
Реализация GET эндпоинтов для получения списка данных.
Пагинация и базовая фильтрация.
Оценка: Корректность JSON-ответов.
Неделя 8: REST API – Продвинутый уровень (5%)
Реализация поиска (Full-text search) и сложной фильтрации.
Подключение Swagger UI (Auto-docs).
Оценка: Удобство использования API и качество документации.
Неделя 9: Аутентификация и Безопасность (5%) 
Реализация JWT авторизации для доступа к API.
Управление переменными окружения (Secrets).
Оценка: Невозможность доступа к защищенным методам без токена.
Неделя 10: Асинхронность и Оптимизация (5%) 
(Опционально) Внедрение RabbitMQ/Redis для фоновых задач.
Оптимизация SQL-запросов и индексов.
Оценка: Архитектурная грамотность.
Неделя 11: CI/CD Pipeline (5%) 
Настройка GitHub Actions.
Автоматический прогон линтеров и сборки контейнеров.
Оценка: Зеленый статус пайплайна в репозитории.
Неделя 12: Тестирование (Unit & Integration) (5%) 
Написание тестов для API и парсера.
Покрытие кода (Coverage) > 40%.
Оценка: Количество и качество тестов.
Неделя 13: Нагрузочное тестирование (10%) 
Написание скриптов для JMeter/k6/Locust.
Эмуляция высокой нагрузки (100+ одновременных запросов).
Оценка: Предоставление отчета с графиками RPS и Latency.
Неделя 14: Мониторинг и Деплой (10%) 
Настройка Prometheus + Grafana (дашборд состояния сервиса).
Финальная сборка docker-compose для продакшн-режима.
Оценка: Наглядность метрик и простота запуска.
Неделя 15: Финальная защита (Demo Day) (10%) 
Демонстрация работы Сервиса в реальном времени.
Показ собранной базы данных.
Анализ результатов нагрузочного тестирования.
Оценка: Комплексная работа системы.

Итоговое распределение оценки 
Оценка за командную работу (70%) — функциональность продукта, качество кода и DevOps практик.
Индивидуальная оценка (30%) — личный вклад (количество коммитов, Pull Requests) и ответы на защите.
